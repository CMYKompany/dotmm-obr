# HANDOFF — DotMM Level 1 Owlbear Rodeo Importer

Companion to `/CLAUDE.md` (read that first). This file carries the longer
narrative: how the data was produced, what was tried and rejected, exact
regeneration commands, and the debugging timeline behind the open issues.

## 1. Project goal

Seamless import of Dungeon of the Mad Mage Level 1 into Owlbear Rodeo from
owned assets: six high-res sub-maps (A–F) with dd2vtt VTT files (walls,
doors, lights, embedded 100 px/cell PNGs), GM overlay images (room-number
labels), the module text, and the DotMM Companion. One import produces a
playable scene: map, dynamic fog, doors, lights, GM room labels + notes,
monster tokens (matched by name to the user's own token pack in their OBR
library), secret doors, and teleport markers.

User-facing decisions already made:
- All-in-one custom extension (not a wrapper around existing importers).
- Token art is supplied by the user; the pipeline matches by filename.
- Sub-maps import either individually or merged into ONE combined scene
  (user currently uses the combined path with TP/transparent variants).
- Token placement is anchor-based ("right room, sensible spot"), not
  author-exact; occasional manual nudging is accepted.

## 2. Content pipeline (already run; outputs in /packs)

The pipeline ran against source assets NOT included in this repo
(copyright): six `Level1_<L>_BG_<WxH>_FVTT.dd2vtt`, six GM overlay PNGs,
the printed book map, module text, Companion text.

Stages (scripts in /pipeline, in order):
1. `annotate_clusters.py` — connected-component clusters on overlay PNGs
   → `packs/clusters.json` (positions of every label glyph, grid coords).
2. `ocr_indexed.py` — per-cluster tesseract OCR (8↔3↔6 confusions common).
3. `curated_anchors.py` — the authoritative hand-curated cluster→label
   table (all ambiguities resolved) → `packs/anchors_curated.json`.
   Notable resolutions: C#1='18' (not 13), C#6='20', E#8='35',
   E#14/15='28b'/'28c', rotated-S glyphs are secret doors, and map D
   carries TWO '27' labels — the alcove and the floating demiplane room
   (variant field distinguishes them; they form a teleport pair).
4. `register_bookmap.py` + `global_fit.py` — OCR of the printed book map +
   robust uniform-scale fit of the global frame to book pixels; validated
   the anchor set (median residual < 3 cells, 11 correspondences kept).
5. `build_content.py` — merges anchors with hand-written room data
   (names, monster rosters with counts/notes, short paraphrased GM notes,
   teleports) → `packs/content_A..F.json` + `packs/token_manifest.json`.
6. `refine_origins.py` — attempted wall-vector seam verification.
   RESULT: tiles share no wall geometry in overlaps (match fractions
   9–34% ≈ noise); origins therefore remain label-derived, rounded to
   integers. Do not retry unconstrained wall/image correlation — both
   were attempted (`global_frame.py` phase correlation, `wall_register.py`
   Hough voting) and failed for structural reasons (no shared art,
   axis-aligned aliasing).

Regenerating `content.js` (repo root) after editing packs:

```python
import json
packs = {L: json.load(open(f'packs/content_{L}.json')) for L in 'ABCDEF'}
manifest = json.load(open('packs/token_manifest.json'))
out = ("// Generated by the DotMM content pipeline - do not edit by hand.\n"
       "export const DOTMM_PACKS = " + json.dumps(packs, indent=1) + ";\n\n"
       "export const TOKEN_MANIFEST = " + json.dumps(manifest, indent=1) + ";\n")
open('content.js', 'w').write(out)
```

## 3. Extension architecture (v1.1.2)

Data flow, single import:
```
dd2vtt file(s) → import.js
  ├─ extractFog()            walls/lights/doors in GRID CELLS (dpi-agnostic)
  ├─ base64ToFile()          embedded PNG → File (baseMap)
  ├─ buildMerged()           per-map builders at global ORIGINS; every item
  │                          gets K.cell = intended position in cells
  ├─ controllerItems()       fog JSON chunked ≤8KB across hidden text items
  └─ OBR.assets.uploadScenes([scene])
Scene opened → background.js
  ├─ syncFromScene()         read controller + fog chunks; state.dpi = getDpi()
  ├─ reconcile()             ONE-TIME (per reconciledDpi): measure map bounds
  │                          via getItemBounds → rescale/move maps onto the
  │                          grid lattice; snap all K.cell items to cell×dpi;
  │                          verify-log residuals
  └─ rebuildLocal()          local Wall items (fog walls + CLOSED doors) and
                             Light items (map lights SECONDARY, vision PRIMARY),
                             signature-gated to skip no-op rebuilds
```

Metadata keys (common.js `K`): controller, fogChunk, door, room, vision,
cell, mapImage, localTag. Plugin id `com.dotmm.importer`.

Combined-scene specifics: baseMap = first map alphabetically; other maps
must first be pushed to the user's library (panel step 2b: upload → pick
back). Seam connectors between included maps are dropped as noise.

## 4. Debugging timeline (why the code looks the way it does)

1. v1.0.0 relative manifest paths → DNS error (`host.tldimport.html`).
   OBR concatenates naively. → absolute URLs (v1.0.1).
2. Long manifest description broke installation (user-discovered).
3. "Import failed: undefined" — uploadScenes rejects with non-Error;
   scene.grid I assembled was likely invalid. → fmtError(), staged errors,
   grid/fog fallback tiers (v1.0.x), later grid omitted entirely.
4. Alignment triple-failure (grid/tokens/walls): dpi hints unreliable.
   → K.cell metadata + background reconciliation (v1.1.0).
5. Background crash `MissingDataError: No scene found` from
   scene.local.getItems with no scene open → reconcile never ran.
   → ready-guards + try/catch on every async entry point (v1.1.1).
6. Reconcile formula (`imagePx × sceneDpi/gridDpi × scale`) mismatched
   observed rendering on one import (native pixels) but matched on the
   next (dpi honored) — OBR is inconsistent → measure with getItemBounds
   instead of computing (v1.1.1).
7. Fractional label-derived ORIGINS put maps B–F off the grid lattice
   → integer origins (v1.1.1). D y-origin 54 vs 55 unresolved (54 chosen).
8. v1.1.1 log verified maps moved EXACTLY onto the 150-lattice
   (deltas = cells×50 for every map, scale 1.0), yet user still reported
   misalignment → v1.1.2 added verify-logging and the Re-align button;
   awaiting the user's verify residuals + a named misaligned pairing.
   This is the live open issue. See CLAUDE.md "Open issues".
9. Token bug: "Bandit" matched the "Bandit Captain" image when no exact
   "Bandit" file existed (substring tie) → length-proximity tiebreak in
   tokenMatchScore (v1.1.2).

## 5. Immediate next steps

1. Get from the user: `[DotMM] verify map …` residual lines after
   Re-align, plus one close-up naming the misaligned pairing
   (grid↔map / tokens↔map / walls↔map). Each pairing isolates a distinct
   subsystem.
2. If tokens↔map: inspect K.cell values on a misplaced token vs its room's
   grid in content.js (possible stale scene or metadata stripping).
3. If walls↔map: dump one wall chunk's cells and compare against the
   dd2vtt polygon for a known corridor.
4. If grid↔map: check OBR grid offset/type in scene settings; the grid
   itself may carry a non-zero offset.
5. Longer term: per-map nudge UI (arrow buttons moving a map ±1 cell and
   its items with it) would absorb the ±1-cell seam uncertainty; Levels
   2+ need the pipeline rerun against their assets.

## 6. Environment notes for Claude Code

- Static site, no build step, no package.json. Syntax check:
  `cp X.js /tmp/X.mjs && node --check /tmp/X.mjs` (extension files are at
  the repo root).
- Deployed via GitHub Pages from repo root; the manifest hardcodes
  `https://cmykompany.github.io/dotmm-obr/` — update all four URL fields
  if the repo moves.
- OBR caches manifests: bump version AND remove/re-add the extension.
- Runtime verification requires a human in an OBR room; design changes to
  be observable via `[DotMM]` console logs.
